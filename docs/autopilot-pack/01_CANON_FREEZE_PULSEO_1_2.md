# 01 - Canon fige Pulseo 1.2

## 1. Decision officielle

**Pulseo 1.2** devient le **canon produit unique**.

Cela implique :

- plus de refonte generale du systeme ;
- plus de concurrence entre documents historiques ;
- toute nouveaute passe par addendum, lot, ADR ou backlog ;
- les archives restent consultables, mais non souveraines.

## 2. Ce que le canon fige

### 2.1 Positionnement

Pulseo est une **plateforme modulaire** a facade simple, alimentee par des moteurs internes plus profonds. Le visible doit rester lisible, calme et actionnable.

### 2.2 Moteurs internes figes

Les moteurs de reference sont :

- **Obscura** - memoire ;
- **Nexus** - liens et graphe ;
- **Prisme** - facettes, priorites et policies ;
- **Forge** - sorties materialisees ;
- **Loic** - orchestration ;
- **DeepSense** - modulation et lecture implicite ;
- **Trust Layer** - consentement, revue, audit ;
- **Runtime & Ops** - exploitation et production.

### 2.3 Principes non negociables

- facade produit simple ;
- profondeur interne cachee mais traçable ;
- explicabilite minimale avant recommandation sensible ;
- consentement adapte au niveau de risque ;
- lineage pour tout artefact ;
- sandbox avant promotion de contenu ou regle sensible ;
- capacite de rollback.

## 3. Ce que le canon n'est pas

Le canon n'est pas :

- un depot d'archives total ;
- un manifeste conceptuel infini ;
- la branche VPS a elle seule ;
- un document unique qui remplace les specs d'execution.

## 4. Ce qui alimente le canon sans le remplacer

### 4.1 Archives conceptuelles

AION-COSMOS, GENESE-X, documents d'emergence, exports d'historique, sandboxes anciennes.

Usage :
- inspirer ;
- enrichir le lot 33 ;
- fournir vocabulaire, patterns, embryons, simulations.

### 4.2 Branches techniques

Pulseo Terminal IA, runtime auto-heberge, scripts VPS, stacks d'execution.

Usage :
- alimenter l'infra et l'ops ;
- raccorder au produit ;
- servir d'implementation partielle.

## 5. Regles de changement du canon

### 5.1 Changements acceptes

- correction de coherence ;
- ajout de lot ;
- ajout d'ADR ;
- precision technique necessaire a l'execution ;
- fermeture d'ambiguite bloquante.

### 5.2 Changements refuses

- reouverture complete de l'architecture ;
- remplacement implicite du canon par un doc satellite ;
- ajout d'un nouveau vocabulaire total sans mapping vers l'existant ;
- gonflement indefini du perimetre mobile MVP.

## 6. Statut cible des familles documentaires

| Famille | Statut | Action |
|---|---|---|
| Pulseo 1.2 | Canon | Geler |
| Archives conceptuelles | Reference | Indexer |
| Branches VPS | Technique | Raccorder |
| Historique conversationnel | Source | Ingerer |
| Sandbox / simulation | Extension | Formaliser |

## 7. Resultat attendu

Tout le travail futur doit pouvoir repondre a deux questions simples :

1. est-ce aligne avec Pulseo 1.2 ;
2. est-ce rattache a un lot, un livrable ou un ADR precis.
