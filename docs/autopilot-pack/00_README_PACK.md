# Pulseo 1.2 - Pack d'execution autopilote

## Objet

Ce pack fige **Pulseo 1.2** comme source canonique, puis applique dans l'ordre les recommandations de pilotage :

1. gel du canon ;
2. registre maitre d'avancement ;
3. specification mobile reliée au VPS ;
4. design system canonique ;
5. pack d'execution junior ;
6. lot 33 pour la bibliotheque semantique sandboxée.

## Regle centrale

A partir de ce pack :

- **Pulseo 1.2** est la source de verite produit ;
- les branches historiques nourrissent, mais ne remplacent pas le canon ;
- toute evolution passe par un lot, un ADR, un backlog item ou un module sandbox ;
- le prochain objectif produit n'est pas une nouvelle vision, mais une **release Android publiable connectée au VPS**.

## Mode d'emploi

Lire dans l'ordre.

- Le fichier 01 fixe le cadre.
- Le fichier 02 dit ou en est le projet.
- Le fichier 03 dit ce qui doit etre livre cote app Android et VPS.
- Le fichier 04 rend le design executable.
- Le fichier 05 transforme le canon en kit de developpement junior.
- Le fichier 06 formalise la bibliotheque semantique et le bac a sable meta-neuromimetisynaptique.
- Le fichier 07 met l'ensemble en sequence de travail.

## Decision deja prise

Le projet ne repart pas dans une refonte doctrinale. Il entre dans une phase de **clarification, outillage et livraison**.
