# 04 - Design System canonique Pulseo

## 1. Objet

Transformer l'intention produit de Pulseo en systeme d'interface executable, coherent et industrialisable.

## 2. Intentions design

Pulseo doit paraitre :

- simple ;
- calme ;
- rassurant ;
- premium ;
- contemporain ;
- doux sans etre mou ;
- profond sans etre esoterique ;
- motivant sans pression agressive.

## 3. Principes design

### 3.1 Clarite d'abord
L'utilisateur doit comprendre quoi faire en quelques secondes.

### 3.2 Respiration
L'interface doit laisser de l'air. Pas de saturation visuelle.

### 3.3 Chaleur controlee
Le produit peut etre humain et chaleureux sans tomber dans le decoratif excessif.

### 3.4 Priorite au geste utile
L'utilisateur doit pouvoir agir vite : check-in, SOS, rituel.

### 3.5 Profondeur cachee
Le systeme peut etre tres riche, mais l'ecran doit rester sobre.

## 4. Tokens a definir

### 4.1 Couleurs
- palette primaire ;
- palette secondaire ;
- neutres ;
- succes ;
- avertissement ;
- erreur ;
- info ;
- etats disables.

### 4.2 Typographie
- police primaire ;
- echelle de tailles ;
- graisses ;
- hauteur de ligne ;
- styles titres / corps / labels / captions.

### 4.3 Espacements
- grille 4 ou 8 px ;
- marges ecran ;
- espacements composants ;
- zones tactiles minimales.

### 4.4 Formes
- rayons ;
- cartes ;
- champs ;
- boutons ;
- modals ;
- pills ;
- chips.

### 4.5 Ombres et elevation
- niveaux legers ;
- pas d'effet lourd ;
- hierarchie discrete.

### 4.6 Motion
- transitions courtes ;
- feedbacks utiles ;
- pas d'animation decorative gratuite.

### 6.1 Racines
- Accueil
- Check-in
- Rituels
- Phoenix
- Profil

### 6.2 Acces persistant
- SOS visible ;
- notifications ;
- reglages.

## 7. Etats UX obligatoires

Chaque ecran doit definir :

- vide ;
- chargement ;
- succes ;
- erreur ;
- offline ;
- contenu bloque ;
- contenu obsolete.

## 8. Direction de contenu

### 8.1 Ton
- simple ;
- direct ;
- chaleureux ;
- non medicalisant ;
- non moralisateur.

### 8.2 Hierarchie texte
- titre clair ;
- sous-titre utile ;
- CTA lisible ;
- message d'erreur actionnable.

## 9. Accessibilite

- contrastes suffisants ;
- tailles lisibles ;
- cibles tactiles confortables ;
- navigation comprehensible ;
- labels de champs clairs ;
- etats non relies a la couleur seule.

## 10. Livrables design a produire

- page de direction artistique ;
- table des tokens ;
- bibliotheque de composants ;
- maquettes des ecrans MVP ;
- guide de navigation ;
- guide d'accessibilite ;
- comportements motion.

## 11. Definition of Done du design system

Le design est considere suffisamment mature quand :

- tous les ecrans MVP reutilisent les memes tokens ;
- les composants sont nommes et consistants ;
- les etats UX sont definis ;
- la navigation est stable ;
- l'equipe peut construire sans improviser l'apparence.
