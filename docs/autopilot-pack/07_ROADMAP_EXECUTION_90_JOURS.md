# 07 - Roadmap d'execution 90 jours

## 1. Objet

Mettre l'ensemble en sequence praticable sur 90 jours, sans rouvrir la vision.

## 2. Phase 1 - Gel et cockpit (J1 a J10)

### Objectifs
- geler Pulseo 1.2 ;
- publier le registre maitre ;
- classer canon / branches / archives.

### Livrables
- decision de gel ;
- registre maitre ;
- index documentaire ;
- backlog reclassé.

## 3. Phase 2 - Mobile et design (J11 a J30)

### Objectifs
- spec mobile ;
- design system ;
- screen flows ;
- perimetre release store.

### Livrables
- spec mobile ;
- design system ;
- maquettes MVP ;
- checklist Play Store.

## 4. Phase 3 - Pack junior (J31 a J50)

### Objectifs
- produire les artefacts reels de transmission.

### Livrables
- OpenAPI ;
- schemas events ;
- bootstrap repo ;
- env ;
- seeds ;
- DoR / DoD ;
- QA checklist.

## 5. Phase 4 - Build MVP relié au VPS (J51 a J75)

### Objectifs
- coder et integrer la premiere release utile.

### Livrables
- auth ;
- accueil ;
- check-in ;
- cravings ;
- SOS ;
- Phoenix ;
- rituels ;
- monitoring ;
- staging.

## 6. Phase 5 - Lot 33 MVP (J76 a J90)

### Objectifs
- lancer la bibliotheque semantique minimale.

### Livrables
- registre d'objets ;
- recherche semantique ;
- liens de base ;
- review queue ;
- simulation/review minimale ;
- rapport d'audit.

## 7. Regles de pilotage pendant les 90 jours

- aucune reouverture du canon sans decision explicite ;
- toute idee nouvelle va en sandbox ou backlog ;
- priorite au livrable publiable ;
- le design suit le MVP ;
- le lot 33 ne retarde pas la release mobile MVP.

## 8. Definition de succes a 90 jours

Le succes est atteint si :

- le projet sait ou il en est ;
- le canon est stable ;
- l'app Android MVP est specifiée et en cours de release ;
- le VPS est raccorde ;
- le design est executable ;
- l'equipe junior peut avancer sans oral massif ;
- le lot 33 est lance en MVP.
