# Code of Conduct

## Our standard

We want PulseoCore to be a respectful, rigorous, and useful open-source project.

Examples of behavior that help:

- respectful disagreement;
- constructive feedback;
- clear technical reasoning;
- empathy toward users and contributors;
- documenting decisions instead of gatekeeping them.

Examples of unacceptable behavior:

- harassment or intimidation;
- personal attacks;
- discriminatory language;
- repeated bad-faith disruption;
- sharing private or sensitive information without consent.

## Project culture

This repository works best when contributors favor:

- clarity over posturing;
- traceability over ambiguity;
- small reviewable steps over chaos;
- open documentation over hidden context.

## Enforcement

Maintainers may remove, edit, or reject contributions, comments, or interactions that violate this code of conduct.

## Reporting

For conduct problems, use the security/reporting channel described in `SECURITY.md` or open a private maintainer contact route when available.
