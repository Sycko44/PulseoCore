# Contributing to PulseoCore

Thanks for contributing.

## First principles

Please keep contributions aligned with the repository baseline:

- open source first;
- self-hostable core;
- reproducible setup;
- no hidden mandatory proprietary dependency;
- clear documentation for every meaningful change.

## Before opening a large contribution

For major changes, open an issue or discussion first and clarify:

- the problem being solved;
- the impacted area;
- the expected user or system benefit;
- whether the change affects the Pulseo 1.2 canon.

## Contribution types

You can contribute through:

- documentation;
- bug fixes;
- tests;
- design system improvements;
- developer tooling;
- mobile/VPS/runtime features;
- semantic sandbox and library work.

## Pull request expectations

Each PR should contain:

- a clear summary;
- the scope of the change;
- screenshots or payload examples when useful;
- notes about risks or migrations;
- documentation updates if behavior changed.

## Keep changes reviewable

Prefer:

- one concern per PR;
- small to medium diffs;
- explicit naming;
- reproducible setup steps.

## Documentation rule

If a change affects behavior, contracts, architecture, runtime, or UX, update the matching docs.

## Security and sensitive features

For anything related to auth, consent, privacy, safety, review, or sandbox promotion, be extra explicit and link the decision back to the relevant docs.

## Code of conduct

By participating, you agree to follow the repository code of conduct.
